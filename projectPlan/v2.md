# Crime Scene Cleaner
Game Design Document — Post MVP Expansion

Version: 2.0  
Project Status: Post-MVP Development  
Engine: Expo / React Native  
Perspective: Top-Down  

---

# 1. Game Vision

Crime Scene Cleaner is a **cleanup mystery game** where the player restores chaotic environments while uncovering the story behind what happened.

The game blends:

- satisfying cleaning mechanics
- environmental storytelling
- light puzzle discovery
- short episodic levels

The player does not simply clean.

The player **reconstructs events** through the environment.

---

# 2. Core Gameplay Loop

The main gameplay loop evolves from simple cleaning into discovery.

Initial Loop (MVP)

enter room  
→ clean stains  
→ reach 100% clean  
→ next level  

Expanded Loop

enter messy room  
→ observe environment  
→ clean objects  
→ discover clues  
→ understand the story  
→ finish cleanup  
→ reveal before/after scene  

---

# 3. Player Fantasy

The player fantasy is:

"I arrive after chaos and restore order."

The player acts as:

- a cleaner
- an observer
- an accidental investigator

The satisfaction comes from:

- transforming chaos into order
- discovering clues
- completing the environment

# 4. Core Mechanics

## Cleaning Mechanics

Primary actions:

- mop stains
- collect trash
- repair objects
- reposition furniture
- dispose debris

Each action increases **Cleanliness Score**.
clean stain → +10
collect trash → +5
repair object → +15

---

## Tool System

Tools create gameplay variety.

| Tool | Function |
|-----|------|
Mop | large area cleaning |
Brush | small precision cleaning |
Spray | dissolves stains |
Repair Kit | fixes objects |
Trash Bag | collects debris |

Tool selection becomes part of strategy.

# 26. Event System Design

The Event System introduces dynamic moments during cleaning.

Events create variety and prevent repetitive gameplay.

Event Categories

Environmental Events
- leaking pipe creates new stain
- power flicker darkens room
- falling object creates debris

Character Events
- pet runs through room leaving footprints
- neighbor knocks and interrupts

Chaos Events
- object falls from shelf
- glass shatters creating new debris

Event Trigger Rules

Events should trigger based on:
- time spent in level
- number of cleaned objects
- random probability

Example Event Flow

player cleans stain
→ event triggers
→ new mess appears
→ player must react

This ensures levels remain dynamic and unpredictable.

---

# 27. Advanced Level Variety

To avoid repetition across episodes, levels should vary in structure.

Variation Types

Large Room Levels
- many small messes

Focused Puzzle Levels
- few but complex mess interactions

Time Pressure Levels
- limited time cleanup

Investigation Levels
- many hidden clues

Chaos Levels
- frequent events and spreading mess

Mixing these types across episodes maintains engagement and pacing.
